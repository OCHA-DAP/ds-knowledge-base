# Afghanistan Drought — 2026-04-04

## Summary

The 2026 Afghanistan drought anticipatory action framework triggers pre-positioned humanitarian assistance for up to 23 prioritised districts across five northern and western provinces (Balkh, Faryab, Jawzjan, Sar-e-Pul, Badghis) if spring drought is forecast or detected. It uses a two-window OR trigger: Window A fires in early March if the ECMWF SEAS5 MAM precipitation forecast falls at or below a 1-in-6-year return-period threshold (≤129 mm for the merged five-province area); Window B fires in early April if a composite drought indicator (CDI) combining five z-scored ERA5/FAO/SEAS5 indicators exceeds a 1-in-4-year threshold (CDI ≥ 0.503). The first window to fire releases the full $22 million combined CERF–AHF pre-allocated budget (CERF $12M + AHF $10M). This is a revised, endorsed successor to the April 2025 framework (its pilot season), updated following an after-action review of the first-ever activation.

## Method

**Data flow:** OCHA/CHD monitors two independent indicator streams:

- **Window A (March):** SEAS5 March-issued seasonal forecast of March–April–May precipitation, retrieved from the OCHA prod PostgreSQL database (fed directly by ECMWF). Province-level values are area-weighted across the five merged provinces to produce a single regional value (mm). Compared against the empirical RP6 threshold (128.96 mm, loaded from blob parquet).

- **Window B (April):** Five raw indicators (ASI, VHI, mixed observed/forecast precipitation, ERA5 snow cover, ERA5 1-m soil moisture) are retrieved from FAO GIEWS (ASI, VHI), ECMWF ERA5-Land (snow cover, soil moisture, March precip), and ECMWF SEAS5 (April and May forecast precip). Each is area-weighted across the five provinces, z-score standardised using 1984–2025 historical distributions (sign-flipped so positive = drought stress, except ASI which is kept as-is), then combined via a ridge-regression-optimised weighted mean to produce the CDI. CDI ≥ 0.503 is the trigger.

Both workflows are implemented as GitHub Actions manual-dispatch pipelines (`w1_seas5_monitor.yaml`, `w2_cdi_monitor.yaml`) that run Python scripts, upload plot + JSON summary to Azure Blob (currently the **dev** blob slot — `stage='dev'`), and email notifications via Listmonk to a distribution list. SEAS5 forecasts are read from the **prod** PostgreSQL DB; all blob I/O (thresholds, weights, outputs) is against the dev slot. Both workflows default to `test=true` and `email_group=core_developer`, so a live activation notification requires the operator to override those defaults. ERA5 data ingestion is a prerequisite step for Window B.

## Trigger logic

- **Keys off:** Window A — ECMWF SEAS5 MAM seasonal precipitation forecast (March-issued). Window B — CDI composed of FAO ASI & VHI, ERA5 snow cover, ERA5 1-m soil moisture, and a mixed observed (ERA5 March precip) + forecast (SEAS5 April, May) precipitation indicator.
- **Decision rule (plain language):** In early March, CHD computes the area-weighted average MAM precipitation forecast for the five merged provinces. If it is at or below 129 mm (RP6 at the merged level), Window A fires and the full allocation is released. If Window A did not fire, CHD computes the CDI in early April from five z-scored indicators (using optimised ridge weights). If CDI ≥ 0.503 (RP4 at the merged level), Window B fires and the full allocation is released. If neither fires, no disbursement occurs for that season.
- **Activation structure:** OR logic across two sequential windows; first window to trigger releases the entire budget. No partial activation.
- **Calibration:** Thresholds and CDI weights were derived from 42 years of historical data (1984–2025). The outcome variable for model training is whether the end-of-season ASI exceeds its empirical 1-in-4-year RP. Window A: tested multiple configurations; pre-season observational indicators (Feb snow cover, soil moisture, cumulative precip) did not add skill over SEAS5 alone. Window B: ridge regression selected CDI component weights; validated with leave-one-out cross-validation (LOOCV), achieving F1 = 0.82 (9 of 10 historical drought years identified, 2 false activations). The overall framework activates approximately 1-in-4 years (i.e., at least one window fires in roughly 10–11 out of 42 years).
- **Authoritative source:** 2026 endorsed PDF (`framework_doc`). Technical annex (Section 8) is the most precise specification.
- **Operated by:** null — CHD runs the monitoring pipeline and sends activation notifications. No external operator.

## Monitoring

CHD monitors both windows on the relevant dates each year (early March for W1, early April for W2). The pipeline is:
1. ERA5 monthly data download (`ERA5 Land Monitor` GHA workflow, prerequisite for W2).
2. Dispatch `SEAS5 W1 Monitor (2026)` in early March.
3. Dispatch `CDI W2 Monitor (2026)` in early April (after ERA5 download).

Each workflow can email CHD, OCHA Afghanistan, the AA Group, CERF, and AHF with a plot and trigger status — but only if dispatched with `email_group=full_list` and `test=false`; the shipped defaults are `core_developer` + `test=true`. Results are uploaded to Azure Blob at `ds-aa-afg-drought/monitoring_outputs/{year}/`, on the **dev** blob slot (`stage='dev'`). Monitoring cadence is annual (spring season only). The cron schedules in both workflows are commented out, so each season's run must be dispatched manually. There is no live dashboard currently deployed.

## Key decisions & rationale

- **Merged five-province AOI (changed from 2025):** The 2025 framework used a per-province OR-any-province logic across three provinces (Faryab, Sar-e-Pul, Takhar). The 2026 redesign merged five provinces (adding Balkh and Badghis, dropping Takhar and Badakhshan) into a single area-weighted regional index. This both tightens geographic focus to the cluster with strongest co-occurrence of drought and sets overall RPs at the AOI level rather than the province level — aligning the stated RP with actual activation frequency more cleanly.
- **Full allocation on first trigger (changed from 2025):** The 2025 framework split the CERF budget ($3.4M for Window A, $6.6M for Window B). The 2026 revision releases the full allocation ($22M) upon whichever window fires first, on the basis that split funding created coordination complexity and that the windows serve the same operational period.
- **Ridge regression weights for CDI (changed from 2025):** The 2025 CDI used equal-ish weights chosen by iterating over all 5%-increment weight combinations and selecting by F1 (the grid-search approach in `weight_selection.md`). The 2026 revision uses ridge regression coefficients (via LOOCV), which were found to produce better cross-validated F1 (0.82, identifying 9/10 drought years with 2 false activations).
- **Window A changed from February to March issue:** In 2025, Window A used the February-issued SEAS5 forecast. In 2026, it uses the March-issued forecast, giving one additional month of observational data before the decision and a somewhat narrower lead time but higher forecast skill.
- **Increased budget and pooled fund coverage:** Total pre-agreed financing rose from $20M (2025) to $22M (CERF $12M + AHF $10M). AHF funding is now formally integrated into the framework allocation; in 2025, AHF supported Window B only.
