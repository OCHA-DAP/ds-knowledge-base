# Afghanistan Drought — 2026-04-04

> The canonical trigger is the code at `code_ref`; this page explains it, it does not redefine it.

## Summary

The 2026 Afghanistan Drought Anticipatory Action Framework is a two-window, OR-logic trigger backed by $22 million in pre-arranged funding (CERF $12M + AHF $10M). It covers five spring-rain-dependent northern provinces (Balkh, Faryab, Jawzjan, Sar-e-Pul, Badghis) and activates if either of two seasonal decision windows fires. Window A (March) evaluates the ECMWF SEAS5 MAM precipitation forecast against a 1-in-6-year return period threshold; Window B (April) evaluates a ridge-regression Combined Drought Indicator (CDI) of five z-scored observational and forecast inputs against a 1-in-4-year threshold. The first window to trigger releases the entire $22M pre-arranged allocation. This is the second endorsed version; the 2025 pilot activated Window B in April 2025 and triggered the full allocation.

## Method

Data flows from three sources: ECMWF SEAS5 seasonal forecasts (retrieved from the CHD prod postgres database), ERA5/ERA5-Land monthly zonal statistics (ingested via GEE into Azure blob by a separate ERA5 Land Monitor GHA workflow), and FAO GIEWS ASI and VHI (fetched live from FAO public URLs). For Window A, the SEAS5 March-issued MAM (March–April–May) forecast is aggregated to a single area-weighted regional value across the five provinces and compared against the RP6 threshold (128.96 mm / ~129 mm). For Window B, five indicators are each z-scored using a 1984–2025 baseline, inverted where needed so that a high z-score always indicates poor agricultural conditions, then combined via a weighted average (ridge regression weights). The resulting CDI scalar is compared against the threshold 0.503. All monitoring is implemented in Python (`src/monitoring_2026/`) and executed manually via GitHub Actions workflows (cron schedules are commented out; dispatch is manual). Results are emailed to the AA Group via Listmonk transactional API.

## Trigger logic

- **Keys off:** Window A — ECMWF SEAS5 MAM precipitation forecast (March issuance). Window B — Combined Drought Indicator comprising FAO ASI and VHI (March dekad 3), ERA5-Land snow cover and 1-metre soil moisture (March), and blended observed/forecast precipitation (ERA5 March obs + SEAS5 April–May forecast issued in April).
- **Decision rule (plain language):** In early March, CHD retrieves the SEAS5 March forecast and computes an area-weighted MAM precipitation total for the five provinces. If that total falls at or below 129 mm (RP6), Window A fires and the full $22M is released immediately. If Window A does not fire, CHD waits until early April when ERA5 March data and FAO March dekad-3 observations become available. The five CDI component indicators are z-scored, weighted, and averaged into a single CDI value. If CDI ≥ 0.503 (RP4), Window B fires and the full $22M is released.
- **Activation structure:** Two independent windows with OR logic. The first to exceed its threshold releases the entire allocation (no partial releases across windows). If Window A activates, Window B is not evaluated operationally.
- **Calibration:** Thresholds derived from 42 years of historical data (1984–2025). Drought years defined as those where FAO end-of-season ASI exceeds its 1-in-4-year empirical RP. Window A threshold calibrated on SEAS5 MAM forecast per the merged 5-province AOI (~1-in-6-year RP). Window B CDI weights estimated via ridge regression; threshold optimized for F1 score using leave-one-out cross-validation (LOOCV F1 = 0.82, 9/10 historical drought years captured, 2 false activations). The combined (either window) activation frequency is approximately 1-in-4 years.
- **Authoritative source:** The 2026 framework PDF (`framework_doc`). The repo (`code_ref`) implements it.
- **Operated by:** OCHA Centre for Humanitarian Data (CHD). CHD monitors both windows and notifies OCHA Afghanistan CO, AA Group, CERF, and AHF by email upon threshold breach.

## Monitoring

CHD monitors both windows via two Python scripts run as manual-dispatch GitHub Actions workflows:

- **Window A** (`w1_seas5_monitor.yaml`, ~March 5): Dispatched when the SEAS5 March forecast is available. Script `w1_monitor_seas5_transactional.py` queries SEAS5 from the prod postgres database, computes area-weighted MAM total for the five provinces, compares against RP6 threshold from blob, plots a bar chart of the historical series, and sends a Listmonk transactional email to the AA distribution list.
- **Window B** (`w2_cdi_monitor.yaml`, ~April 10): Requires a prerequisite ERA5 data download step (`era5_land_monitor.yaml`, dispatched after ERA5 March data becomes available ~April 7). Script `w2_monitor_cdi_transactional.py` loads ERA5 data from blob, fetches FAO ASI and VHI from public URLs, queries SEAS5 April-issued forecasts from prod postgres, computes the CDI, and sends an analogous Listmonk email.

Monitoring artifacts (PNG plot + JSON summary) are uploaded to Azure blob (`ds-aa-afg-drought/monitoring_outputs/{year}/`) on the **dev** stage. The full operational runbook is in `.github/MONITORING.md` on the `2026-monitoring-w2` branch.

## Key decisions & rationale

- **Five provinces (vs. three in 2025):** Balkh, Faryab, Jawzjan, Sar-e-Pul (Northern Region) and Badghis (Western Region) selected based on: high share of rain-fed agricultural land, prolonged drought history (2020–2023 anomalies + 2025 impacts), and strong inter-province correlation. Takhar (in the 2025 design) was dropped; Jawzjan, Balkh, Badghis added. Provinces are treated as a single merged cluster for trigger evaluation — no per-province activation.
- **Merged (single) AOI vs. per-province (2025 design):** The 2025 pilot triggered per province with different CDI thresholds. This was replaced with a single merged regional threshold to reduce operational complexity and better capture coherent drought conditions across the cluster.
- **Full allocation on first trigger (vs. split across windows in 2025):** In 2025, CERF funds were split between Window A and B. For 2026, any single window activation releases the entire $22M. This simplifies disbursement and accelerates response.
- **Ridge regression CDI weights:** The CDI component weights were estimated via ridge regression (leave-one-out cross-validated) rather than equal or heuristic weighting. This optimises sensitivity to drought years as defined by FAO end-of-season ASI. LOOCV F1 = 0.82 for the April CDI.
- **SEAS5 forecast only for Window A (no pre-season obs):** Testing showed that adding February observational indicators (snow cover, soil moisture, cumulative precipitation) to SEAS5 did not improve and actually reduced skill (F1 = 0.58 vs. 0.47). Window A uses the SEAS5 MAM forecast as a standalone indicator.
- **CERF + AHF dual pooled funds:** Afghanistan is one of two globally with both CERF and AHF backing an AA framework. CERF covers UN agencies; AHF covers national and international NGOs (10 partners). Activities are designed to be complementary with no duplication.
- **Validity period:** Two years (2026–2027). If the framework activates in 2026, lessons from implementation will be applied for the 2027 revision.

## Changes from previous version

The 2025 pilot framework (publication anchored to 2025-04-01 per the repo's `pub_mo_date`; no separate 2025 framework PDF was ingested and the exact ERC-endorsement date is not evidenced in the repo or 2026 PDF — see Open questions; superseded by this page) used:
- **3 provinces** (Faryab, Sar-e-Pul, Takhar) → expanded to **5 provinces** (Balkh, Faryab, Jawzjan, Sar-e-Pul, Badghis).
- **Per-province CDI thresholds** → single **merged AOI threshold** (0.503).
- **Split allocation** across Windows A and B → **full $22M on first trigger**.
- **CDI weights:** 2025 used manually-specified or earlier design weights (version "20250401"); 2026 uses ridge regression weights re-estimated with the expanded 5-province feature set and updated 1984–2025 historical record.
- **Funding:** $22M total in 2026 (CERF $12M + AHF $10M); the 2025 framework had similar overall structure but AHF was not pre-arranged at the same level.
- **Monitoring pipeline:** 2025 used R scripts (`src/trigger_monitoring/`); 2026 uses Python (`src/monitoring_2026/`) with Listmonk transactional email and GitHub Actions, driven by the `2026-monitoring-w2` branch.

Changes were driven by the After-Action Review workshop convened in early 2026 following the 2025 Window B activation, incorporating lessons from CERF-AHF and WFP after-action reviews and the ongoing FAO impact evaluation.

## Open questions / known issues

- **PDF footnote RP discrepancy:** Footnotes 3 and 4 cite per-province empirical return periods (Window A: 4.2yr, Window B: 5.2yr) that contradict the headline merged-AOI claims (Window A: ~1-in-6yr, Window B: ~1-in-4yr). The inversion (Window A headline > footnote; Window B headline < footnote) is unexpected and should be queried with the trigger sub-group. The merged AOI RP can differ from per-province RP, but the direction of the discrepancy is unusual.
