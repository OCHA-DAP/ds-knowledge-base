# Central America Dry Corridor Drought — 2025-02

> **Superseded by [2026-03-13](2026-03-13.md)** — the 2026–2027 renewal (expanded Guatemala scope, El Salvador moved to Ahuachapán/Santa Ana, SEAS5-only trigger). This page is the historical Feb-2025 version.

> The canonical trigger is the code at `code_ref`; this page explains it, it does not redefine it.

## Summary

The Dry Corridor Anticipatory Action Framework covers El Salvador, Guatemala, and Honduras (Nicaragua was de-scoped from the 2025 update). It triggers on seasonal precipitation forecasts falling below a 1-in-4-year return period threshold for two cropping-season windows: Primera (May–August, monitored March–May) and Postrera (September–November, monitored June–August). CERF has pre-arranged $10.5M USD for disbursement to FAO, WFP, UNICEF, and WHO/PAHO upon trigger activation in any of the three countries. The framework was endorsed in February 2024 and updated in February 2025 (re-posted to the OCHA portfolio on 29 July 2025). No triggering occurred during 2024. **This version is superseded by the [2026–2027 renewal](2026-03-13.md)**, which expanded scope and switched Guatemala to a SEAS5-only trigger; the activation history (GTM Postrera 2025, GTM+SLV Primera 2026) is recorded there.

## Method

Each month during monitoring periods, OCHA CHD downloads the latest ECMWF SEAS5 forecast from the Copernicus Data Store, aggregates seasonal rainfall (MJJA for Primera or SON for Postrera) across selected admin-1 units in each country using a pixel-weighted mean, and compares the result to pre-computed empirical thresholds derived from the 1981–2022 historical hindcast at RP=4.

For Guatemala, INSIVUMEH (the national meteorological service) provides a deterministic seasonal forecast in NetCDF format. CHD receives this manually from INSIVUMEH contacts (between the 5th and 10th of each month) and uploads it to Azure Blob. The monitoring run is therefore split: a preliminary email (ECMWF only, excluding Guatemala) goes on ~6th; a final email adding Guatemala's INSIVUMEH-based status follows once the file is uploaded. If INSIVUMEH data is not available by leadtime 0 (May for Primera), ECMWF SEAS5 is used as the fallback for that month only.

Thresholds are stored in a versioned parquet file in blob (`ds-aa-lac-dry-corridor/framework_update_2025/`) and read by the monitoring code at runtime. The active threshold file as of June 2025 is `df_thresholds_seas5_insivumeh_adm1_refined_insiv_update_202506.parquet`.

## Trigger logic

- **Keys off:** ECMWF SEAS5 seasonal precipitation forecast (CDS); INSIVUMEH deterministic national forecast (Guatemala only, leadtimes 1–4 and Postrera all leadtimes).
- **Decision rule (plain language):** At each monitoring month, compute the area-weighted mean seasonal rainfall forecast across the selected admin-1 units for each country. If that value is ≤ the pre-computed RP-4 empirical threshold for that country/season/leadtime/forecast-source combination, the trigger is met for that country. Each country is assessed independently — one can trigger while others do not.
- **Activation structure:** Two windows, both independently activatable per country. Window A monitors Primera (MJJA) during March–May. Window B monitors Postrera (SON) during June–August. In-country validation meeting convened within 2 days of CHD alert; CERF disbursement follows RC/HC endorsement.
- **Calibration:** Empirical percentile from 1981–2022 hindcast (42 years). Linear interpolation between ranked observations to return the value at the 25th percentile (RP=4). Done separately per country, window, leadtime, and forecast source to avoid cross-source bias.
- **Authoritative source:** The 2025 umbrella framework PDF and per-country annexes (`framework_doc`). The repo (`code_ref`) derives and implements the thresholds.
- **Operated by:** OCHA CHD runs the live monitoring and sends the activation email; the repo is the production monitoring system.

## Monitoring

Monthly manual run by CHD DS analyst. On ~6th of month: download ECMWF CDS forecast → run `src/01_download_latest_ecmwf_cds.R` → run `src/02_monitor_forecast.R` (or equivalent 2025 script `src/monitoring_2025/update_activation_status.R`). A preliminary email goes without Guatemala; once INSIVUMEH files are received and uploaded to blob (`raster/raw/insivumeh/raw/v202506/...`), analyst re-runs to produce the final email with Guatemala status.

A GHA workflow exists (`.github/workflows/cadc_2025_monitoring.yaml`) but is `workflow_dispatch`-only (no cron) and runs `update_activation_status.R` against the **dev** blob slot (`DS_AZ_BLOB_DEV_SAS`) with the prod DB. INSIVUMEH data retrieval itself is semi-manual (email from INSIVUMEH contact, analyst uploads to blob under `stage="dev"`). See README for the current file naming convention.

## Historical activations

No activation occurred during 2024 (the 2025 country annexes confirm "during 2024, none of the agreed upon trigger thresholds for either the Primera or Postrera seasons were reached"). The framework's first activations came **after** this version — GTM Postrera in July 2025 and GTM+SLV Primera in March 2026 — and are recorded on the current version, [2026-03-13](2026-03-13.md). Historical backtesting (1981–2022) shows the RP-4 threshold would have been exceeded in various years (flagged red in repo plots).

## Key decisions & rationale

**Guatemala uses INSIVUMEH, not ECMWF, as primary source.** INSIVUMEH is the official national forecaster for Guatemala; the framework mandates "national forecast whenever possible/available; ECMWF otherwise." INSIVUMEH provides deterministic forecasts at leadtimes 1–4 months but cannot produce a same-month (leadtime 0) forecast — so May Primera monitoring uses ECMWF SEAS5 for Guatemala only.

**RP-4 threshold (not RP-5 or RP-3).** The CERF mandate was "1-in-4-year return period". Analysis confirmed this gives a reasonable activation frequency (~25% per year per country, lower joint probability). The team chose RP-4 over a stricter threshold to maintain sufficient detection sensitivity during strong El Niño years.

**Admin-1 sub-national scope.** The 2025 update refined the AOI from national-level to specific admin-1 units with highest vulnerability and alignment to IPC Phase 3+ areas. This reduced false positives from national averaging masking local deficits. Nicaragua was excluded partly because its admin-1 drought signal was weaker.

**Baseline years 1981–2022.** The year 2023 (strong El Niño, drought conditions across the region) was excluded from threshold calibration to avoid thresholds being artificially lowered by including an extreme year that was itself a crisis — this is explicitly noted in the code (`filter(year(issued_date) <= LAST_BASELINE_YEAR)` with `LAST_BASELINE_YEAR <- 2022` in analysis/05a_create_thresholds_geographically_refined_2025.R).

**ECMWF MARS → CDS migration.** In April 2024, the operational forecast data source shifted from ECMWF MARS (direct API) to Copernicus Data Store (CDS) for operational accessibility. Threshold absolute values were recalibrated to account for any bias between MARS and CDS versions of SEAS5 (see analysis/03_ECMWF_MARS_Vs_CDS.qmd).

## Changes from previous version

**Supersedes:** `2024` — the original 4-country framework (SLV, GTM, HND, NIC — endorsed February 2024). This page is the February-2025 update (re-posted to the OCHA portfolio on 29 Jul 2025 with no content change). Key changes from 2024:
- **Nicaragua removed** from the formal endorsed scope. The 2024 framework included Nicaragua as a 4th country (its specific 2024 CERF allocation is not confirmed in the extracted source PDFs). The 2025 framework covers only 3 countries ($10.5M total: $4M GTM, $4M HND, $2.5M SLV). Nicaragua monitoring continues informally.
- **Sub-national AOI refined.** Trigger now keyed to specific admin-1 units (Chiquimula for GTM; El Paraíso/Francisco Morazán for HND; San Vicente for SLV) rather than national-level averages.
- **INSIVUMEH formally integrated** as the primary forecast source for Guatemala (leadtimes 1–4). The 2024 framework mentioned national forecasts generically; the 2025 monitoring code and threshold files explicitly operationalize INSIVUMEH.
- **El Niño framing removed.** The 2024 PDFs were written specifically in the context of the 2023–2024 El Niño event; the 2025 update removes El Niño-specific framing to make the framework applicable to any severe drought year.
- **Funding clarified:** CERF $10.5M pre-arranged for 2024–2025 two-year window. Co-financing from ECHO (~0.9M EUR), WFP internal (1–1.5M USD), FAO internal (1M USD) explicitly acknowledged in the 2025 umbrella document.

## Open questions / known issues

- Nicaragua: the decision to de-scope was made operationally but is not explained in the 2025 PDF. Should CHD publish a separate Nicaragua framework page or note the change formally?
- The 2025 CERF pre-arrangement was for 2024–2025 (two years from Feb 2024 ERC endorsement). As of July 2025, the framework is near the end of its pre-arrangement window. What happens if a trigger fires in the Postrera 2025 window (August monitoring)?
- The INSIVUMEH data file naming convention changed in June 2025 (new `v202506/` subdirectory prefix); the blob loading code was updated accordingly (`insivumeh.R`), but any analyst running an older branch would load the wrong files.
- StartNet is exploring a separate sub-framework for Guatemala (Quiché, Baja Verapaz — different admin-1 units than Chiquimula); see `origin/startNet` branch and `analysis/07a_StartNet_create_thresholds_geographically_refined.R`. No published framework for this yet.
