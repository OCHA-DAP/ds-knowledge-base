# Yemen Flood — 2023

> The canonical trigger is the code at `code_ref`; this page explains it, it does not redefine it.

## Summary

The Yemen flood anticipatory action (AA) framework is a 2023 pilot designed to preemptively release funds for IDP populations in high flood-hazard sites across two governorates — Hajjah and Marib — ahead of flood events. The trigger is based on a 3-day cumulative rainfall forecast from CHIRPS-GEFS: if the 3-day cumulative sum within a 7-day forecast window crosses a 2-year return period threshold for either Area of Interest (AOI), the trigger activates for that AOI and a pre-agreed YHF allocation ($3M) is released to implementing partners. The framework was endorsed and monitored for the 2023 rainy season but was not activated; monitoring has since been disabled.

## Method

The analysis pipeline proceeds as follows:

1. **Impact data baseline:** CCCM flood report data (2021–2022) for high flood-hazard IDP sites in Hajjah and Marib was used to characterize flood events and identify priority areas.
2. **AOI definition:** Convex hulls were drawn around high-risk IDP sites (REACH flood hazard score = "High Hazard") within target districts (Abs in Hajjah; Ma'rib and Ma'rib City in Marib), buffered by 5 km. Two AOI iterations were explored — full governorate hulls and district-level hulls — and the district-level definition was adopted for monitoring.
3. **Historical calibration:** CHIRPS daily rainfall (1981–2022 or 2000–2022 filtered) was extracted as a zonal mean over both AOI definitions and converted to 3-day rolling sums. GEV-based return period functions were fitted separately for each AOI and governorate to obtain 2-year return period (RP) levels.
4. **CHIRPS-GEFS calibration:** CHIRPS-GEFS forecasts (2000–2022, excluding 2020) were processed analogously (3-day rolling sums per date-of-issue). 2-year RP levels were computed at each leadtime over leadtimes 3–10 (leadtimes 1–2 excluded — insufficient 3-day rolling-sum values) and compared with the CHIRPS historical RP. (Note: a code comment in `email.Rmd` says `mean(lt3:7)`, but the RP derivation in `exploration/10_CHIRPS_GEFS.Rmd` actually spans leadtimes 3–10 — see `discrepancies`.)
5. **Threshold selection:** For each governorate, the minimum (more conservative) 2-year RP estimate across the two AOI calculations was rounded down to the nearest integer: **Hajjah = 24 mm**, **Marib = 20 mm**.
6. **Forecast monitoring:** A GitHub Actions workflow ran every 3 hours, downloading the latest CHIRPS-GEFS daily forecast, computing zonal means over the AOI, rolling to 3-day sums across leadtimes 1–10, and checking whether any day in the 7-day trigger window (leadtimes 1–7) crossed the threshold. Results were emailed daily to a recipient list.

## Trigger logic

- **Keys off:** CHIRPS-GEFS daily precipitation forecast; 3-day cumulative zonal mean over AOI convex hull + 5 km buffer
- **Decision rule (plain language):** For each governorate AOI independently, compute the 3-day rolling sum of the CHIRPS-GEFS daily forecast. If any forecast day within the 7-day trigger window (leadtimes 1–7 from today) produces a 3-day cumulative rainfall >= the governorate threshold, the trigger fires for that AOI.
- **Activation structure:** Two independent spatial windows (Hajjah AOI, Marib AOI). Either can trigger independently. There is no AND condition between them.
- **Calibration:** Threshold = 2-year return period of 3-day cumulative rainfall, calibrated jointly against CHIRPS historical (2000–2022, excluding 2021 where incomplete) and CHIRPS-GEFS hindcast (2000–2022 excluding 2020, leadtimes 3–10). The more conservative (lower) of the two RP estimates was taken for each governorate.
- **Authoritative source:** No published framework PDF was located. The internal technical note is a private Google Doc referenced in the README. `trigger_source` is therefore `repo`. The code at `src/email/email.Rmd` and `src/update_trigger.R` implements the live trigger logic.
- **Operated by:** OCHA Centre for Humanitarian Data / CHD Data Science (GHA-based, Google Drive storage). Monitoring was run via GitHub Actions and has since been disabled.

## Monitoring

Monitoring was run via GitHub Actions (`yem-trigger-monitoring.yaml`) on a 3-hourly schedule during the 2023 rainy season. The workflow downloaded the latest CHIRPS-GEFS daily 16-day forecast from the CHC data server, cropped it to the AOI, computed zonal means, and applied the trigger logic. Results (including a forecast plot with threshold lines) were emailed via SMTP to a distribution list. Intermediate raster TIFFs and zonal CSVs were archived to Google Drive.

As of the 2024 season, the GHA was updated to flag emails as "Post Monitoring" (informational only, not linked to the AA framework) and has since been fully disabled. No live monitoring pipeline is currently operating.

Note that during the active 2023 season the operational alert emails were dispatched with the subject line "Email Test: Yemen AA Rainfall Forecast Monitoring" (hardcoded in `update_trigger.R`) — the live deployment was never promoted out of a test/dev slot before monitoring ended. Only the post-2024 (`is_post_monitoring`) branch uses the "Post Monitoring:" subject. See `discrepancies`.

## Historical activations

The trigger framework was not activated in 2023. The README explicitly states: "The trigger framework was not activated in 2023 and the monitoring has since been disabled." No CERF allocation or YHF disbursement under this specific trigger mechanism has been identified. The $3M YHF reserve allocation was pre-arranged and ready for release, but no rainfall event crossed the thresholds during the monitored period.

## Key decisions & rationale

- **2-year return period:** A relatively frequent threshold was chosen to maximize trigger sensitivity for IDP flood protection, accepting higher false-alarm rates. The CCCM flood impact data (2021–2022) was used to validate that events at approximately this rainfall level coincide with reported shelter damage.
- **3-day cumulative window:** Selected over shorter (1-day) or longer (5–30 day) windows based on performance testing against the CCCM impact database; 3-day accumulation showed reasonable alignment with reported flood events.
- **CHIRPS-GEFS over ECMWF HRES:** Both were evaluated. CHIRPS-GEFS was selected for the live trigger due to data accessibility (public URL, no license barriers) and sufficient skill at 3–7 day lead times. ECMWF HRES hindcast analysis was retained in the repo but is not used in production.
- **District-level AOI over full governorate:** The AOI was narrowed from the full Hajjah/Marib governorate hulls to just the priority response districts (Abs, Ma'rib, Ma'rib City) where YHF partners planned to respond. This produced slightly different RP levels; the lower threshold of the two calculations was adopted to be conservative.
- **Threshold conservatism:** When the district AOI and governorate AOI produced different 2-year RP levels, the lower (more sensitive) threshold was taken for each governorate. This biases toward triggering rather than missing events.

## Open questions / known issues

- No public framework document has been published. The repo README notes "ENDORSED" status but no PDF on ReliefWeb/unocha.org was found. If the framework is to continue or be updated, a formal published document should be produced.
- The GHA monitoring workflow has been disabled. If the framework is reactivated for future seasons, the monitoring pipeline needs to be re-enabled and tested with current CHIRPS-GEFS URL structures.
- The 2023 season did not trigger, so the pre-arranged $3M YHF allocation was not released. It is unclear whether the YHF allocation remains available for future seasons or has lapsed.
- Implementing agencies receiving the YHF funds are not documented in the repo or in publicly accessible sources.
- The CHIRPS-GEFS v1.2 product used in calibration (2000–2022) may have been updated; threshold recalibration against more recent hindcast data would be advisable before reactivation.
- Target population (number of IDP households/individuals in the pilot AOIs) is not captured in the repo; the YHF 1st Reserve Allocation 2023 report cites >300,000 people across a broader $8M allocation (AA + preparedness combined) in 22 districts across 5 governorates — the AA-specific share is not broken out.
