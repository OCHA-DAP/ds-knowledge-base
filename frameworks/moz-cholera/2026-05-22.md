# Mozambique Cholera — 2026-05-22

## Summary

This framework triggers coordinated anticipatory action when weekly AWD surveillance data indicates an abnormal cholera escalation in Mozambique, across two parallel pathways: a district-level pathway (≥2 districts in the same province crossing thresholds for 3 consecutive weeks) and a province-level pathway (provincial total ≥2,500 AWD cases/week for 3 consecutive weeks). An additional gate requires cholera confirmation or probable classification per GTFCC criteria. On activation, CERF releases US$750,000 (up to two activations in two years), split between UNICEF (WASH) and WHO (Health), for immediate multi-sectoral response.

## Method

Weekly epidemiological data on acute watery diarrhea (AWD) cases — collected by MISAU/national health authorities, shared by WHO with CHD — is evaluated every week at district and province level. The analysis combines two retrospectively calibrated indicators:

1. **District indicator:** For each district, cases in a given week are flagged if: (a) cases are at least 4× the previous week's cases, OR (b) cases per 100,000 population exceed the province-specific 99th percentile threshold. In either case, a floor minimum must also be met: ≥100 cases/week for non-capital districts, ≥300 for capital districts. A district must meet these criteria for 3 consecutive weeks to count toward the district trigger.

2. **Province indicator:** Province total AWD cases ≥2,500 for 3 consecutive weeks (without per-district disaggregation).

The 99th percentile thresholds are calculated per province using historical district-level AWD surveillance time series, normalized to cases per 100,000 population (WorldPop 2025 denominators, ADM2 aggregation), excluding zero-reporting weeks to avoid downward bias.

A combined disease confirmation gate is required regardless of which pathway fires: cholera must be confirmed or classified as probable (GTFCC definition). This is a human operational step — it is not implemented in the automated monitoring pipeline.

## Trigger logic

- **Keys off:** Weekly AWD cases by district and province from WHO surveillance data (shared as structured Excel with Semana columns).
- **Decision rule (plain language):** Two parallel pathways. DISTRICT PATH: At least 2 districts in the same province must each, in every one of 3 consecutive weeks, have AWD cases above the floor minimum AND meet at least one of: 4× the previous week's count, or cases per 100k exceeding the province's 99th percentile. PROVINCE PATH: The province total must be ≥2,500 AWD cases/week for 3 consecutive weeks. Either path triggers. A confirmed or probable cholera outbreak (GTFCC criteria) is required in addition to either path.
- **Activation structure:** OR logic between two windows (district-path OR province-path), each itself requiring 3 consecutive weeks; AND gate for disease confirmation.
- **Calibration:** Retrospective analysis of historical AWD surveillance time series; 99th percentile calculated on non-zero observations only to avoid downward bias. Province thresholds normalized by population. Biennial recalibration planned. Return periods: province path ~4.7 years, district path ~7.0 years, combined ~2.8 years.
- **Authoritative source:** Framework PDF (`framework_doc`). The monitoring pipeline (`code_ref`) implements the threshold checks.
- **Operated by:** CHD (Centre for Humanitarian Data) monitors and determines trigger activation. WHO provides weekly data. No external model system (IRI Maproom etc.) — trigger evaluated in-house via the ds-aa-moz-cholera-monitoring pipeline.

## Trigger windows

**Counting rule:** Two distinct activation windows — district-path and province-path. They share the same data source and operate in parallel on each weekly update. A single province can fire via either path.

| window | basis | indicator | threshold | lead time | return period | releases |
|---|---|---|---|---|---|---|
| District path | observational | Weekly AWD cases per district: floor met AND (4× prev week OR ≥99th percentile per 100k) for 3 consecutive weeks; ≥2 districts in same province | Non-capitals: 100 cases/week floor; Capitals: 300 cases/week floor; per-province 99th percentile (e.g. Cabo Delgado 301.2/100k, Maputo City 226.2/100k) | None (observational; acts at time of threshold crossing) | ~7 years | US$750,000 CERF per activation |
| Province path | observational | Weekly province-total AWD cases ≥2,500 for 3 consecutive weeks | 2,500 cases/week | None (observational) | ~4.7 years | US$750,000 CERF per activation |

**Additional AND gate (not a separate window):** Confirmed or probable cholera outbreak (GTFCC criteria). Applies to both windows. A maximum of 2 activations (total US$1.5M) in the 2-year validity period. Six-month cooldown per province prevents re-triggering.

## Monitoring

Weekly automated monitoring via GitHub Actions (Tuesday 08:00 UTC) in `ds-aa-moz-cholera-monitoring`. WHO emails a surveillance Excel file weekly; a team member renames it to start with `Mozambique Cholera Data` and uploads to Azure Blob (`ds-aa-moz-cholera-monitoring/monitoring/`). The pipeline validates the file, parses new Semana columns since the last run, evaluates both trigger paths per province, and sends a summary alert email via Listmonk (list 102) regardless of trigger status. A six-month province-level cooldown is tracked in a JSON state file on blob. No deployed web app — monitoring output is delivered by email.

## Key decisions & rationale

- **Observational over forecast:** Complex cholera prediction models were assessed and explicitly rejected as insufficiently reliable for operational decision-making. The trigger is entirely observational, acting at early signs of abnormal transmission rather than anticipating future spread.
- **Dual pathway:** Province-level path was added drawing on lessons from the DRC cholera AA framework (the first coordinated cholera AA). It provides a safety net when a province-wide surge occurs without necessarily producing district-level hotspots.
- **Floor minimums:** The 100/300 case floor prevents the 4× rule from firing on trivially small case counts (e.g. 1→4 cases would otherwise trigger). Capitals have a higher floor (300) reflecting their larger reporting denominators.
- **99th percentile calibration:** Calculated on non-zero observations only to avoid downward bias from periods of zero reporting. Province-specific (not national) to account for highly variable epidemiological patterns across provinces.
- **3-consecutive-weeks requirement:** Requires sustained elevated transmission to reduce false alarms from single-week reporting noise or spikes.
- **Disease confirmation gate:** Added to further reduce false alarms, given AWD (acute watery diarrhea) is a proxy indicator and not all AWD cases are cholera. Confirmation is sought; probable classification (GTFCC) is accepted when confirmation is not feasible.
- **6-month cooldown:** Prevents repeated activation during a single extended outbreak, preserving the two-activation budget for distinct events.
- **Maputo City special treatment:** All districts summed into a single entity. In the PDF framework, Maputo City operates under the same logic as other provinces (district-level path or province-level path). The monitoring pipeline sums Maputo City districts and applies capital-district logic (floor 300, 4× or 99th percentile, 3 weeks) — consistent with the PDF intent even if the internal comments describe it as "week-on-week increase."

## Changes from previous version

No prior version. This is the first published anticipatory action framework for cholera in Mozambique. The design incorporated lessons from the DRC cholera AA framework, specifically: integration of a provincial-level trigger, greater geographical flexibility, strengthened awareness among implementing partners, and clear coordination mechanisms.

## Open questions / known issues

- **Maputo City trigger documentation vs code:** The README, constants.py comment, and email footer claim "week-on-week increase" logic for Maputo City, but the code applies standard capital-district logic. The comment should be corrected to match the code, or vice versa.
- **Disease confirmation gate:** Not implemented in any automated system. The monitoring pipeline emails trigger status without verifying cholera confirmation. The activation protocol (PDF section 6 table) assigns this step to CHD requesting information from WHO, but this workflow is entirely manual.
- **Dev storage:** STAGE = "dev" and dev SAS tokens are used in the live monitoring pipeline. If a production storage stage is intended, it has not been configured.
- **district_case_limit = 1500:** This constant in the analysis repo appears to be a legacy artefact. Its purpose relative to the final framework is unclear — no analysis notebook uses it for the endorsed trigger logic.
- **Threshold file on blob:** The per-province 99th percentile thresholds are stored in `district thresholds.xlsx` on blob rather than hardcoded in the repo. Version control and auditability of this file is unclear.
- **Winsorisation:** Code for winsorising case counts exists in the analysis notebook but is disabled (both limits = 0). The reason it was not used in the final calibration is not documented.
