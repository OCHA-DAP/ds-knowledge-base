# Burkina Faso Flood — 2025-08

> The canonical trigger is the code at `code_ref`; this page explains it, it does not redefine it. Note that the repo analysis was written using the old administrative division names; the endorsed framework uses new region names.

## Summary

The Plan national d'action anticipatoire pour les inondations au Burkina Faso 2025-2027 (PNAAI) is a government-led anticipatory action plan targeting fluvial flooding in six priority regions. The primary operational trigger for 2025 uses ECMWF SEAS5 monthly precipitation forecasts: if the forecast mean daily rainfall for a given month (July, August, or September) exceeds region-specific 3-year return period thresholds, the FHRAOC-funded response is activated. A secondary fallback trigger uses the CONASUR national alert system (confirmed flood occurrence + IMERG observational precipitation data), enabling an early response within 72 hours when the seasonal forecast threshold is not met. River-gauge-based triggers on the Mouhoun river (Scenarios 1 and 2) were developed for the plan but deferred to 2026. When activated, pre-arranged FHRAOC funds (1 million USD) are released to four partner NGOs to deliver anticipatory actions in sectors including food security, health, nutrition, WASH, shelter, and protection across approximately 79,800 people in 11,400 households.

## Method

1. On the 5th of each month (July, August, September), the OCHA Centre des Données Humanitaires (CDH) reads the SEAS5 forecast issued for that month.
2. The forecast mean daily precipitation over each of the five target regions is compared to pre-calculated 3-year return period thresholds derived from historical SEAS5 data (calibrated on 1993–2024).
3. If any region's forecast exceeds its monthly threshold, the trigger is considered activated for that region. Activation is independent per month and per region (non-cumulative).
4. CDH notifies the GTTAA secretariat (SE-CNSA, OCHA, PAM, FAO), which informs the Humanitarian Coordinator and relevant ministers; FHRAOC funds are then released to pre-identified partners.
5. In parallel, CONASUR monitors for confirmed flood events; if a CONASUR alert is issued (with IMERG validation), a rapid early response is activated within 72 hours regardless of whether the SEAS5 threshold was reached.
6. At minimum 20% of the budget is reserved for preparedness activities to ensure operational readiness before the trigger fires.

## Trigger logic

- **Keys off:** SEAS5 monthly mean daily precipitation forecast (Scenario 3); CONASUR alert system + IMERG observed precipitation (Scenario 4); Mouhoun river gauge readings from DGRE at Yaran, Ténado, and Pâ stations (Scenarios 1 and 2 — deferred to 2026).
- **Decision rule (plain language):** Each month from July to September, if SEAS5 forecasts mean daily rainfall above the 1-in-3-year threshold for any of the five regions, the trigger fires for that region. The forecast is issued on the 5th of the month, giving a 0–25 day action window before month end. Activation does not carry over: if August triggers and funds are spent, September monitoring still occurs, but there is no remaining budget. If SEAS5 does not fire but CONASUR reports a confirmed flood in a covered region, the fallback Scenario 4 releases the funds for an early response within 72 hours.
- **Activation structure:** Single-stage. Either SEAS5 threshold OR CONASUR observational alert fires the fund release. Scenarios 3 and 4 run simultaneously during the rainy season. Whichever fires first consumes the budget.
- **Calibration:** 3-year return period, calibrated using SEAS5 historical data (equal issued and valid month — i.e. 0-month lead time). Thresholds set per region and per month. Repo analysis uses `rp_calc.calculate_groups_rp` with Weibull plotting positions.
- **Authoritative source:** The latest framework PDF (`framework_doc`), validated August 2025. The repo (`code_ref`) implements the calibration analysis for the SEAS5 trigger; it predates the final administrative reclassification and covers only 2–5 of the 5 target regions.
- **Operated by:** OCHA CDH monitors and announces the SEAS5 trigger (Scenario 3). CONASUR announces the observational trigger (Scenario 4). The repo is the analysis layer, not a deployed production system.

- **Trigger taken from:** `framework_doc` (August 2025 PDF). The repo provides the calibration analysis for the SEAS5 component (Scenario 3) but does not implement Scenarios 1, 2, or 4.
- **Repo completeness:** `{analysis: partial, deployed_code: lost}` — the SEAS5 return-period calibration analysis is present and matches the PDF methodology; however, (a) no code exists for the river-gauge or CONASUR/IMERG triggers, (b) the analysis uses old administrative names (Sahel, Centre-Nord) not the new names in the endorsed plan, and (c) there is no deployed monitoring pipeline.
- **Discrepancies:** See frontmatter `discrepancies` for the full tagged list. Key issues: (a) repo uses old admin divisions and no old->new region crosswalk is recorded; (b) Scenarios 1 and 2 (Mouhoun gauge) have no live code — `exploration/river_level.md` analyses a *different* gauge (Koriziena, Sahel), not the Mouhoun stations; (c) Scenario 4 (CONASUR + IMERG) has no repo code; (d) several **PDF-internal** inconsistencies — the Financement section cites a non-existent "Scénario 5", Scenario 4 states both 48h and 72h for the early-response window, and the region count is given variously as 5 / 6 / 7 (Soum excluded) with province counts of 14 vs 22; (e) the September 2025 activation was characterised as a "fausse activation" in the AAR — the SEAS5 trigger fired but floods did not materialise and activities ran out of season, prompting a top recommendation to integrate local hydrometric data into the trigger.

## Monitoring

The CDH (OCHA Centre des Données Humanitaires) is responsible for monitoring the SEAS5 trigger (Scenario 3) each month from July to September, retrieving forecasts from ECMWF charts. CONASUR monitors flood occurrence for Scenario 4. There is no deployed automated pipeline in the repo; monitoring appears to be manual/ad hoc. The framework calls for results to be communicated to SE-CNSA, SP/CONASUR, OCHA, PAM, FAO, FHRAOC, and the Humanitarian Coordinator.

## Historical activations

The trigger fired on **8 September 2025** (Scenario 3, SEAS5 threshold) — this is the first activation of the framework. However, the After-Action Review (February 2026) classified this as a "fausse activation": the anticipated flooding did not materialize, and administrative disbursement delays meant that partner activities (by APIL, OXFAM, ASMADE, OCADES-Dori) were implemented out of season. Beneficiaries were reported to have asked: "Les pluies sont passées, pourquoi mener les actions maintenant?" (The rains have passed, why are actions being taken now?). Activities nonetheless proceeded under the "no-regret" principle and delivered WASH, sensitization, and targeted distributions. The AAR identified the SEAS5 model's limited integration of local hydrological and pluviometric data as a key reliability concern for future calibration.

## Key decisions & rationale

- **Return period of 3 years**: The plan targets extreme flooding events with a minimum 3–5 year recurrence. A 3-year RP was chosen as the per-region, per-month calibration point (Scenario 3), resulting in a combined national trigger period that differs from 3 years due to multiple independent regional tests.
- **0-month lead time (valid = issued date)**: The SEAS5 trigger uses same-month forecasts (e.g. August forecast used for August threshold), giving only 0–25 days of action window. This is constrained by the fast-onset nature of fluvial flooding, unlike drought frameworks that use 1–3 month lead times.
- **Spatial scope**: Five regions prioritized based on cross-analysis of poverty incidence, resilience capacity index (ICR), flood exposure density, catchment flood areas, and security constraints. The Soum region was excluded due to security access constraints despite meeting eligibility criteria.
- **Phased trigger development**: River-gauge triggers (Scenarios 1 and 2) are acknowledged as more locally grounded but were not ready for 2025 deployment; SEAS5-based trigger (Scenario 3) was used as the operational mechanism for the first year, with a commitment to integrate gauge-based triggers in 2026.
- **FHRAOC covers only 2 of 5 SEAS5 regions**: The 1M USD pre-arranged funding targets Liptako and Koulsé specifically (500k each); the remaining three regions (Bankui, Yaadga, Djoro) are in the endorsed framework scope but lack pre-arranged funding as of the 2025-08 version.
- **20% preparedness floor**: At least 20% of budget must be spent on preparedness activities regardless of activation scenario, to ensure operational readiness.

## Changes from previous version

This is the first version of the PNAAI (no superseded version). The framework document notes it complements the existing anticipatory action plan for drought (piloted by OCHA, funded by CERF). The Croix-Rouge Burkinabè also runs a parallel early-action protocol for fluvial flooding under the IFRC DREF mechanism (500,000 CHF), which is referenced in the plan as complementary financing to be capitalized upon.

## Open questions / known issues

1. **Trigger reliability**: The AAR explicitly questions SEAS5 reliability in the Burkinabè context due to lack of local hydrometric/pluviometric data integration. The top priority recommendation is to adjust the trigger by integrating local data sources.
2. **Disbursement delays**: The first activation revealed that administrative delays between trigger firing and fund release can push implementation out of the rainy season, undermining the anticipatory logic. Pre-financing or revolving fund mechanisms are recommended.
3. **Administrative reclassification gap in repo**: The repo analysis uses old Burkina Faso administrative names (Sahel = BF56, Centre-Nord = BF49) while the endorsed framework uses new names. The exact PCODE mapping between old and new divisions should be documented.
4. **Scenarios 1 and 2 status**: The river-gauge triggers (Mouhoun basin) were deferred to 2026. It is unknown whether they have been developed and endorsed since the framework was published.
5. **Funding gap for 3 regions**: Bankui, Yaadga, and Djoro are in the endorsed SEAS5 trigger scope but do not have pre-arranged funding. Whether additional resources were secured is unknown.
6. **n_windows clarification**: The 5 SEAS5 region-specific triggers (Liptako, Koulsé, Bankui, Yaadga, Djoro) are collapsed into 1 window row in the table because they share identical methodology; only per-region threshold values differ. They are not counted as separate windows.
