# Nigeria Flood — 2025-08-11

> The canonical trigger is the code at `code_ref`; this page explains it, it does not redefine it.

## Summary

This framework pre-arranges $7 million (CERF $5 million + NHF $2 million) for flood anticipatory action in Nigeria. It covers two independent trigger components: (1) a riverine flooding forecast trigger keying off GloFAS and Google Flood Hub discharge forecasts at the Benue River upstream of Adamawa state, releasing CERF and NHF funds to 7 LGAs (Fufore, Yola South, Yola North, Girei, Demsa, Numan, Lamurde); and (2) a flash flooding observational trigger using FloodScan-derived flood exposure, which can release only NHF funds as a reprogramming option to 6 BAY-states LGAs (Bama, Bade, Karasuwa, Madagali, Maiduguri, Ngala). The framework targets up to 350,000 people and is valid for two years (2025–2026), with CERF committed through 2026 and NHF expected to be used by end of 2025.

## Method

**Riverine trigger:** GloFAS reanalysis (historical) and ensemble forecast data are retrieved daily from the Copernicus Climate Data Store (CDS) for the Wuroboki station on the Benue River. Google Flood Hub (GRRR model) forecasts are retrieved via the Google Flood Forecasting API for gauge `hybas_1120842550`. Both discharge values are compared against calibrated thresholds. If either source exceeds its threshold — within the respective lead-time window — the riverine trigger fires and CERF + NHF funds are released immediately.

**Flash flooding trigger:** FloodScan SFED satellite flood extent (3-day rolling average) is overlaid with WorldPop 2020 UN-adjusted population at 300m resolution to produce per-LGA flood exposure estimates. These estimates are compared to per-LGA thresholds calibrated to an approximately 4-year return period. If any LGA threshold is exceeded, NHF funds can be reprogrammed.

**Threshold calibration (riverine):** Both model thresholds were calibrated against FloodScan wet-season (July–November) annual maxima as observational ground truth, using a grid search over all (GloFAS rank, Google rank) pairs with an OR-logic trigger evaluated against Floodscan 5-year return period events. The selected "balanced trigger" sets both sources to approximately a 5.4-year implied return period individually; the combined OR-logic gives an overall ~4.5-year return period. Historical lead times of 14–25 days were observed in reanalysis; the operational GloFAS action window is ≤5 days, and Google's effective lead time is 6–7 days (issued with 1–2 day lag, maximum 8-day forecast).

**Threshold calibration (flash flood):** FloodScan exposure thresholds were set to give equal probability of triggering in each LGA, with an overall ~4-year return period.

## Trigger logic

- **Keys off:** GloFAS v4 ensemble mean discharge (CDS `cems-glofas-forecast`) at Wuroboki (G5004); Google Flood Hub GRRR model discharge at gauge `hybas_1120842550`; FloodScan SFED 3-day rolling average flood extent × WorldPop 2020 population.
- **Decision rule (plain language):** The riverine trigger fires if either GloFAS predicts ≥3,132 m³/s at Wuroboki within the next 5 days, OR the Google forecast predicts ≥1,195 m³/s at the Kangli/Wuroboki gauge at any lead time. Either source alone is sufficient ("OR" logic). The reanalysis (leadtime=0) can also trigger. The flash flooding trigger fires if FloodScan-derived flood exposure in any of the 6 BAY LGAs exceeds its per-LGA threshold.
- **Activation structure:** Two independent triggers. Riverine trigger releases CERF + NHF. Flash trigger (observational, near-real-time) releases only NHF under the NHF reprogramming scenarios.
- **Calibration:** Return-period based. Riverine: ~4.5-year combined OR-logic RP (targeting 5-yr RP flood events as required by CERF). Flash: ~4-year RP per LGA.
- **Authoritative source:** The latest framework PDF (`framework_doc`). The repo (`code_ref`) implements the monitoring and detection logic.
- **Operated by:** null — monitoring is operated by OCHA CHD in collaboration with the NiHSA/NiMet-led Trigger Task Team. The live trigger check runs in this repo via GHA (`monitoring.yml`, daily at 20:00 UTC).

## Monitoring

Automated daily monitoring via GitHub Actions (`monitoring.yml`, cron `0 20 * * *`). The pipeline:
1. Retrieves GloFAS forecast and reanalysis data from CDS for Wuroboki station.
2. Retrieves Google GRRR forecast for `hybas_1120842550` via the Google Flood Forecasting API.
3. Stores results to the dev database (`projects.ds_aa_nga_flooding_monitoring`).
4. Generates and saves plots to Azure blob storage.
5. The framework's activation protocol specifies automated emails to the OCHA/NiHSA/NiMet-led Trigger Group (weekly on Mondays, or immediately if the activation/warning threshold is exceeded). This logic is implemented in `pipelines/send_emails.py` (`action`/`informational` templates), BUT `send_emails.py` is **not** invoked by `monitoring.yml` — the scheduled job currently runs only `check_forecasts.py` and `save_plots.py`. The email step is therefore not dispatched by the deployed pipeline. Additionally, `STAGE` defaults to `dev`, which would send TEST emails unless promoted to `prod` (see `discrepancies`).

Flash flooding is monitored separately via the FloodScan exposure monitoring application (`https://chd-ds-floodexposure-monitoring.azurewebsites.net/`) with a 1-day reporting lag.

Two deployed marimo apps provide monitoring visualization (both currently on development slots):

## Historical activations

**Activated 8 September 2025** (first activation, ~1 month after endorsement): GloFAS and Google Flood Hub forecasts exceeded the endorsed riverine thresholds for the Benue River at Wuroboki. The CERF USD 5M + NHF USD 2M envelope (USD 7M) was released; within ~72 hours WFP delivered anticipatory multi-purpose cash transfers to over 83,500 people across four Adamawa LGAs, with ~350,000 people reached across the 7 target LGAs ([Anticipation Hub](https://www.anticipation-hub.org/news/the-un-and-partners-deliver-anticipatory-actions-ahead-of-floods-in-nigeria)).

The `CERF_YEARS` constant in `src/constants.py` ([2013, 2018, 2022]) refers to historically significant flood calibration reference years used in the trigger performance analysis, not framework activations. The 2022 Nigeria floods (which displaced 4 million people) and the 2024 floods predated or were concurrent with the framework development, not activations under this framework.

The PDF notes that based on reanalysis, the framework would have activated in 2024 by August 20 at the latest (if it had been in place), but historical forecast data for 2024 is unavailable so this cannot be confirmed using forecast data.

## Key decisions & rationale

**OR logic for riverine trigger:** Both GloFAS and Google GRRR were found to offer complementary skill. The OR logic maximizes F1 score and detection rate while maintaining lead time; either source alone can trigger. This also provides redundancy against data-source disruption.

**Dual-source calibration (~5.4 yr each, ~4.5 yr combined):** CERF requires targeting 5-year return period floods. Because the OR logic lowers the combined activation threshold, both sources were calibrated to ~5.4 yr individually so the combined result targets the ~4.5 yr CERF requirement.

**GloFAS 5-day operational window:** GloFAS forecasts extend to day 5 (with day 1 = the forecast issue date). The GloFAS reanalysis can also trigger (leadtime=0). The maximum Google effective lead time is 6–7 days (8-day forecast minus 1–2 day issuance lag).

**Flash flood trigger (NHF only):** Flash flooding is observational (FloodScan SFED) — no reliable forecast source exists at the LGA level. Per-LGA thresholds were set to equalize triggering probability across LGAs. The trigger releases only NHF funds; CERF is reserved for riverine (forecast-based) triggers only.

**NHF reprogramming flexibility:** The NHF $2M is designed to be flexible: if the riverine trigger doesn't fire by mid-September, funds can be reprogrammed to flash-flood response or broader humanitarian priorities (4 explicit scenarios in the framework doc).

**Geographic targeting:** The 7 riverine LGAs in Adamawa state were selected through a four-tier analysis: historical flood/impact data (NiHSA/NiMet), historical flood risk (NEMA), historical average population exposed (FloodScan), and severity of needs (2025 HNRP PiN figures). The 6 flash-flood LGAs were selected based on NEMA flash flood risk mapping and NGO implementation capacity.

## Changes from previous version

This is the first endorsed version of the Nigeria flood framework. The 2024 NHF allocation ($2M, no-regrets strategy for Bama and Numan LGAs) was a precursor pilot, not a formal AA framework with defined triggers. This framework is the first to define explicit, endorsed trigger thresholds and pre-arrange CERF funding.

## Open questions / known issues

1. **Google gauge naming:** The PDF labels the Google gauge "Kangli" but the HYBAS ID (`hybas_1120842550`) is named `WUROBOKI_HYBAS` in the codebase. This discrepancy needs clarification — "Kangli" may be the Google FloodHub display name for the same physical location.

2. **Warning tier not in PDF:** The monitoring code implements a warning tier (GloFAS 2800, Google 1000) that triggers weekly emails to partners. This tier and its thresholds are not described in the endorsed framework document. Should this be formalized or documented as an operational add-on?

3. **Benue state extension:** The `govt-2026-prep` branch contains active analysis for extending the framework to Benue state (Makurdi station). No thresholds are defined yet (`glofas_thresh` and `google_thresh` are `None` in `STATE_CONFIG["Benue"]`). This will require a new framework version when complete.

4. **Production slot deployment:** Both Azure apps serve from `-development-` slugs. These should be promoted to production slots to reflect the endorsed status.

5. **Lead time claimed vs. operational:** The PDF cites 14–25 days lead time from historical reanalysis, while the operational GloFAS window is 5 days and Google's effective window is 6–7 days. The discrepancy should be clarified in future versions.

6. **GloFAS reforecast issue frequency:** The Wuroboki GloFAS reforecast is issued approximately twice-weekly (~35 issue dates per wet season vs ~120 for a daily product), limiting skill evaluation sample sizes.
