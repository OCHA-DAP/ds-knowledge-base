# Data Science Knowledge Base

**The single searchable home for the OCHA Centre for Humanitarian Data, Data Science team's
methods, frameworks, pipelines, and infrastructure** — the institutional knowledge that
otherwise lives in people's heads, scattered repos, and PDFs.

This is a friendlier view of the [knowledge-base repository](https://github.com/OCHA-DAP/ds-knowledge-base) —
exactly the same content, just easier to browse. Use the **search box** at the top to find anything.

<div class="grid cards" markdown>

-   :material-map-marker-radius: **[Framework status map](map.md)**

    Every Anticipatory Action framework on a map — active, recently triggered, in development —
    with triggers, funding, and activation history.

-   :material-table-large: **[Trigger statistics](triggers.md)**

    Published triggers, windows, return periods, and pre-arranged financing across the whole AA
    portfolio — one table per framework, with activations flagged.

-   :material-file-document-multiple: **[Frameworks](frameworks/)**

    The AA frameworks and every version: design, triggers, the rationale behind each.
    Start with the **[catalog](catalog.md)** for a filterable index.

-   :material-pipe: **[Pipelines](pipelines/)**

    The living operational systems — data ingests, monitoring, exposure — and their runbooks.

-   :material-monitor-dashboard: **[Apps](apps/)**

    The deployed dashboards and interactive surfaces.

-   :material-chart-line: **[Analysis](analysis/)**

    Regional overviews, ad-hoc activations, and pre-framework exploration.

-   :material-cog: **[Infrastructure](infrastructure/)**

    Storage, databases, the shared libraries, the dependency graph, and the live pipeline registry.

-   :material-lightbulb: **[Methods](methods/)**

    Cross-cutting how-we-work — the trigger-pattern typology and more.

-   :material-book-open-variant: **[Docs](docs/)**

    How the KB itself works: the ingestion conventions, the design decisions, the roadmap, and
    the public/internal privacy model.

</div>

---

!!! tip "How to use this"
    **Just search.** The box at the top searches every page — framework names, hazards, country
    codes, trigger details, pipeline names, anything. Each page also links back to its source
    (the published PDF, the GitHub repo) so you can always go deeper.
