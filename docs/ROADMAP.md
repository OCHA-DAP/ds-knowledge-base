# Roadmap & status

Living status of the KB build. Update the checkboxes, the "Now / Next" line, and the **Ingestion progress** table as work lands. Rationale for the phasing is in [DESIGN.md](DESIGN.md).

**Now:** the corpus is substantially built and **self-maintaining** (D50). Every content type is ingested — frameworks, pipelines, apps, analysis, libraries — plus the cross-cutting layers (dependency graph + blast radius, the `methods/trigger-patterns` typology, the live pipeline registry) and the daily/weekly automation that keeps it current (drift, PDF-freshness, infra-drift, discovery → `kb-ingest` PRs — see [automation.md](../infrastructure/automation.md)). Scope is now the **full OCHA/CERF AA portfolio**, not just DS-built frameworks (D51), so the open frontier is *comprehensiveness* — historical pilots (som-drought, ssd-flood, mdg-plague, mwi-drought) drafted from public sources, surfaced by `aa-watch`. **Phase 8 started** — blob data inventory (`assets/` content type); pilot GloFAS page for `ds-aa-nga-flooding`.
**Next:** the **front door** (Phase 6 — read-only DB/blob MCP tier on top of the live connector) and **Drive content** (Phase 7c/7e — extraction + slide captions into the private repo, ongoing). For Phase 8: FloodScan coverage parser (cross-project), GeoGLOWS (NPL), CHIRPS-GEFS (MOZ storms), a generic parquet-inventory mode, and the NGA analysis-outputs catch-all.
